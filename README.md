# Provenance Guard

A Flask API that estimates whether submitted text was written by a human or generated by an AI tool. Each submission is scored by two independent signals, combined into a single confidence score, given a human-readable transparency label, and recorded in an append-only audit log. Creators can appeal a classification.

---

## Architecture

```
POST /submit
    │
    ▼
Input validation + cleaning (strip whitespace, reject empty text)
    │
    ▼
Signal 1: LLM Classification (Groq — llama-3.3-70b-versatile)   → llm_score
    │
    ▼
Signal 2: Stylometric Heuristics (pure Python)                  → stylo_score
    │
    ▼
Confidence Scoring:  combined_score = (llm_score * 0.7) + (stylo_score * 0.3)
    │
    ▼
Transparency Label Generation
    │
    ▼
Audit Log Entry written (JSON)
    │
    ▼
JSON response returned to caller
```

The LLM signal is weighted higher (0.7) because it captures semantic properties that pure statistics cannot; the stylometric signal (0.3) is a corroborating structural check.

---

## Detection Signals

### Signal 1 — LLM Classification (Groq)

Assesses semantic and stylistic coherence holistically — whether the text "reads" like AI output (hedged phrasing like "it is important to note", overuse of transitional language, unnaturally even sentence rhythm). Returns a float `0.0–1.0` where `1.0` = high confidence AI-generated.

**Blind spots:** can be fooled by lightly human-edited AI text, or by formal human writing (academic, legal) that stylistically resembles AI output. It reasons purely from surface text and has no access to structural statistics.

### Signal 2 — Stylometric Heuristics (pure Python)

Computes three structural metrics, each normalized to `0.0–1.0` (where `1.0` = AI-like) and averaged into a single `stylo_score`:

1. **Sentence length variance** — AI text has more uniform sentence lengths; human writing mixes short and long. Low variance → more AI-like.
2. **Type-token ratio (TTR)** — `unique_words / total_words`. AI text trends toward lower diversity due to repeated transitional phrases. Low TTR → more AI-like.
3. **Punctuation density** — ratio of punctuation to total characters. AI text clusters around a moderate, consistent density; human writing deviates more.

**Blind spots:** formal human writing scores as AI-like (uniform structure, controlled vocabulary); short texts (under ~80 words) produce statistically unreliable variance estimates; poetry confuses the signal.

## Confidence Scoring

```
combined_score = (llm_score * 0.7) + (stylo_score * 0.3)
```

| Score range | Interpretation |
|---|---|
| 0.75 – 1.00 | Confident the text is AI-generated |
| 0.41 – 0.74 | Uncertain — signals are mixed or the text is ambiguous |
| 0.00 – 0.40 | Confident the text is human-written |

The uncertain band is intentionally wide because a false positive — labeling a human's work as AI — is a more serious error than a false negative on a writing platform. When in doubt, the system says so rather than confidently misclassifying.

---

## Transparency Label Variants

**High-confidence AI (combined_score >= 0.75):**
```
AI-Assisted Content Detected
Our analysis found strong indicators that this content was likely generated
or substantially written by an AI tool. Confidence: [XX]%.
If you believe this is incorrect, you can submit an appeal below.
```

**Uncertain (0.40 < combined_score < 0.75):**
```
Attribution Unclear
Our analysis found mixed signals for this content. We are not confident
whether this was written by a human or generated by an AI tool. Confidence: [XX]%.
If you are the creator, you can submit an appeal to provide more context.
```

**High-confidence human (combined_score <= 0.40):**
```
Human-Written Content
Our analysis found strong indicators that this content was written by a human.
Confidence: [XX]%. Attribution signals are recorded for platform transparency.
```

## API Surface

| Endpoint | Method | Input | Output |
|---|---|---|---|
| `/submit` | POST | `{text, creator_id}` | `{content_id, attribution, confidence, label}` |
| `/appeal` | POST | `{content_id, creator_reasoning}` | `{status, message}` |
| `/log` | GET | — | `{entries: [...]}` |

### Rate Limiting

`POST /submit` is rate-limited to **10 per minute** and **100 per day** per client (in-memory storage via Flask-Limiter). This reflects a realistic single-creator submission frequency while blocking scripted flooding of the LLM signal.

## Setup

```bash
# Install dependencies (uses uv + pyproject.toml)
uv sync

# Provide your Groq API key
echo "GROQ_API_KEY=your_key_here" > .env

# Run the app
uv run python app.py
```

The server starts on `http://127.0.0.1:5000`.

## Usage Examples

### Submit content

```bash
curl -X POST http://127.0.0.1:5000/submit \
  -H "Content-Type: application/json" \
  -d '{"text": "...", "creator_id": "creator-123"}'
```

**AI example** — short AI-generated text scored by both signals:

```json
{
  "content_id": "…",
  "attribution": "uncertain",
  "confidence": 0.63,
  "label": "Attribution Unclear\nOur analysis found mixed signals for this content. …"
}
```

Even though the text was AI-generated, its short length made the stylometric signal unreliable, pulling the combined score into the uncertain band (0.63) rather than a confident AI classification.

**Human example** — informal human writing:

```json
{
  "content_id": "…",
  "attribution": "likely_human",
  "confidence": 0.03,
  "label": "Human-Written Content\nOur analysis found strong indicators that this content was written by a human. …"
}
```

### Appeal a classification

```bash
curl -X POST http://127.0.0.1:5000/appeal \
  -H "Content-Type: application/json" \
  -d '{"content_id": "…", "creator_reasoning": "I wrote this myself; it is a formal essay."}'
```

On appeal, the original audit log entry's `status` is updated from `classified` to `under_review`, and an appeal record (with `content_id`, timestamp, `creator_reasoning`, `original_attribution`, and `original_confidence`) is appended to the log. No automated re-classification is performed — a human reviewer sees both signal scores, the original classification, the confidence score, and the creator's reasoning.

### View the audit log

```bash
curl http://127.0.0.1:5000/log
```

## Known Limitations

- **Formal human writing misclassified as AI.** Academic essays and legal briefs written by humans have low sentence-length variance, controlled vocabulary, and measured punctuation — all of which the stylometric signal reads as AI-like, and which the LLM signal may also score high. This is the most likely false-positive scenario.
- **Short text (under 80 words) produces unreliable stylometric scores.** Sentence-length variance is statistically meaningless with only a couple of sentences, so short content yields a near-meaningless `stylo_score`. The system still runs but operates with low signal quality — as seen in the AI example above, which landed as *uncertain* (0.63) rather than *likely AI*.
